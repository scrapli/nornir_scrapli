basic_netconf_usage
===================

This directory contains a very simple setup to demo basic usage of `scrapli-netconf` with `nornir`.

A very standard `YAMLInventory` lives in the `nornir_data` directory along with a simple `config.yaml` file. There is
 a single host in this inventory -- this host is the IOSXE device in my `scrapli` functional test setup. You can read
  about how to set up that test environment with `vrnetlab` [here](https://github.com/carlmontanari/scrapli#testing) in the `scrapli` documentation. Or you can of
   course just update this inventory to point to a host in your environment for testing!

When using `scrapli-netconf` with `nornir` -- things are pretty much the same as when you use "normal" `scrapli
` -- with one notable and important difference that the `scrapli-netconf` connection options need to live under the
 `scrapli_netconf` key in the connection options dictionary. Here is a simple example from this example dir:
 
```yaml
---
iosxe:
  platform: ios
  connection_options:
    scrapli_netconf:
      port: 830
      extras:
        ssh_config_file: True
        auth_strict_key: False
``` 

With that out of the way, the `demo.py` script here simply runs through a handful of the available netconf tasks
 supported by `scrapli-netconf`. The "result" of these tasks will always be the string output from the NETCONF server
 . The `scrapli_response` attribute of the result (see the last part of this demo script) contains the "original
 " `scrapli-netconf` response object, and all of the normal `scrapli` attributes there, such as start/end/elapsed
  time, raw result, xml_result, etc..
